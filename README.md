# **Содержание**



# **1. Введение**
Данная система представляет собой комплексное решение для анализа орбитальных параметров космических объектов на основе данных TLE (Two-Line Element Set). Система использует методы глубокого обучения для автоматической детекции аномалий и предоставляет детальные рекомендации по коррекции орбиты.
## **1.1. Что такое TLE?**
TLE (Two-Line Element Set) — это стандартный формат данных для описания орбитальных параметров искусственных спутников Земли. Каждая TLE-запись состоит из трёх строк:

- Строка 0: Название объекта
- Строка 1: Каталожный номер, эпоха, производные среднего движения
- Строка 2: Орбитальные элементы (наклонение, RAAN, эксцентриситет и др.)
## **1.2. Основные возможности системы**
- **Парсинг и валидация TLE данных**
- **Автоматическая детекция аномалий** с использованием автоэнкодера
- **Классификация объектов** (спутник, мусор, ступень ракеты)
- **Анализ исторических трендов** изменения орбиты
- **Расчёт углов и величин коррекции** для спутников GEO
- **Оценка рисков** столкновения и входа в атмосферу
- **Контекстно-зависимые рекомендации** для операторов
# **2. Архитектура системы**
Система построена по модульному принципу и состоит из следующих компонентов:
## **2.1. Модули системы**

|**Модуль**|**Описание**|
| :- | :- |
|**parse\_tle.py**|Парсер TLE данных. Извлекает орбитальные параметры, классифицирует объекты, выполняет орбитальные расчёты.|
|**train\_model.py**|Обучение нейронной сети. Содержит автоэнкодер для детекции аномалий и нормализатор данных.|
|**generate.py**|Генерация отчётов. Создаёт детальные текстовые отчёты с рекомендациями, расчётами коррекции и оценкой рисков.|
|**main.py**|Главный скрипт. Координирует работу всех модулей, управляет исторической базой данных TLE.|
## **2.2. Поток данных**
1. **Загрузка TLE файла** (текстовый файл с записями)
1. **Парсинг** каждой записи в числовой вектор (6 параметров)
1. **Нормализация** данных (StandardScaler)
1. **Прогон через модель** (автоэнкодер)
1. **Расчёт ошибки реконструкции** и оценки аномальности
1. **Генерация отчёта** с рекомендациями
1. **Сохранение в историю** для трендового анализа
# **3. Модель машинного обучения**
## **3.1. Архитектура автоэнкодера**
Система использует **вариационный автоэнкодер (VAE)** для детекции аномалий. Модель обучается восстанавливать нормальные орбитальные параметры. Объекты с высокой ошибкой реконструкции считаются аномальными.
### **3.1.1. Энкодер**
Сжимает входной вектор (6 параметров) в латентное пространство (32 измерения):

Input (6) → Dense(128) → BatchNorm → ReLU → Dropout(0.2)

`         `→ Dense(64)  → BatchNorm → ReLU → Dropout(0.2)

`         `→ Dense(32)  → ReLU
### **3.1.2. Декодер**
Восстанавливает исходный вектор из латентного представления:

Latent (32) → Dense(64)  → BatchNorm → ReLU → Dropout(0.2)

`            `→ Dense(128) → BatchNorm → ReLU → Dropout(0.2)

`            `→ Dense(6)   → Output
### **3.1.3. Детектор аномалий**
Дополнительная нейронная сеть на основе латентного представления:

Latent (32) → Dense(32) → ReLU → Dense(1) → Sigmoid → [0, 1]
## **3.2. Параметры обучения**

|**Параметр**|**Значение**|
| :- | :- |
|Функция потерь|MSE (Mean Squared Error)|
|Оптимизатор|Adam (lr=0.001, weight\_decay=1e-5)|
|Batch size|32|
|Количество эпох|100 (с early stopping)|
|Validation split|20%|
|Порог аномалии|0\.01 (настраиваемый)|
## **3.3. Входные данные**
Модель обучается на 6 орбитальных параметрах:

1. **Наклонение (Inclination)** — угол между плоскостью орбиты и экватором [градусы]
1. **RAAN** (Right Ascension of Ascending Node) — долгота восходящего узла [градусы]
1. **Эксцентриситет (Eccentricity)** — степень вытянутости орбиты [безразмерный, 0-1]
1. **Аргумент перигея (Argument of Perigee)** — угол от восходящего узла до перигея [градусы]
1. **Средняя аномалия (Mean Anomaly)** — положение на орбите [градусы]
1. **Среднее движение (Mean Motion)** — количество оборотов за сутки [обороты/день]
# **4. Функциональные возможности**
## **4.1. Классификация объектов**
Система автоматически определяет тип космического объекта по названию из TLE:

- **SATELLITE** — активный или неактивный спутник
- **DEBRIS** — космический мусор
- **ROCKET\_BODY** — ступень ракеты-носителя
- **FRAGMENT** — осколок после столкновения или взрыва
## **4.2. Детекция аномалий**
Аномалия определяется по двум метрикам:

- **Ошибка реконструкции** — среднеквадратичное отклонение между входным и восстановленным вектором. Если превышает порог (по умолчанию 0.01), объект считается аномальным.
- **Оценка аномальности** — выход детектора аномалий в диапазоне [0, 1]. Значения близкие к 1 указывают на высокую вероятность аномалии.
### **4.2.1. Компонентный анализ аномалий**
При обнаружении аномалии система анализирует, какой именно параметр отклонился от нормы:

- Вычисляет абсолютную и относительную ошибку для каждого из 6 параметров
- Применяет весовые коэффициенты важности (эксцентриситет и среднее движение имеют больший вес)
- Определяет главную причину аномалии и формирует диагностическое сообщение
## **4.3. Исторический анализ**
Система ведёт базу истории TLE (до 10 последних записей на объект) и выполняет трендовый анализ:

- **Тренд высоты орбиты**: стабильная, снижающаяся, быстро деградирующая
- **Скорость изменения**: км/день
- **Прогноз входа в атмосферу**: количество дней до схода с орбиты
## **4.4. Расчёт коррекции орбиты**
Для спутников на GEO орбите система рассчитывает требуемые маневры коррекции:
### **4.4.1. Коррекция наклонения (North-South)**
- Требуемое изменение угла [градусы]
- Величина Delta-V [м/с]
- Направление коррекции
### **4.4.2. Коррекция эксцентриситета**
- Изменение эксцентриситета
- Delta-V для перехода к круговой орбите [м/с]
### **4.4.3. Коррекция дрейфа (East-West)**
- Скорость дрейфа [градусы/день]
- Delta-V для стабилизации [м/с]
- Оптимальное время и точка выполнения маневра
## **4.5. Оценка рисков**
### **4.5.1. Риск столкновения**
Система анализирует риск опасного сближения на основе:

- Высоты орбиты (на GEO пояс приходится наибольшая концентрация спутников)
- Эксцентриситета (высокий эксцентриситет увеличивает зону пересечения орбит)
- Наклонения (объекты с разным наклонением пересекают больше орбитальных плоскостей)
### **4.5.2. Риск входа в атмосферу**
Оценивается по:

- Высоте перигея (критический уровень: 120 км)
- Коэффициенту BSTAR (атмосферное торможение)
- Производной среднего движения (скорость снижения орбиты)
# **5. Руководство по использованию**
## **5.1. Установка зависимостей**
pip install torch numpy scikit-learn
## **5.2. Обучение модели**
Для обучения модели на собственных данных:

python train\_model.py

Скрипт загружает данные из data/all\_tle.txt, обучает модель и сохраняет в model/:

- tle\_model.pth — веса модели
- normalizer.pkl — параметры нормализации
- training\_history.json — история обучения
## **5.3. Анализ TLE файла**
Базовое использование:

python main.py --file data/test.txt

Параметры командной строки:

- **--file, -f** — путь к TLE файлу (по умолчанию: data/test.txt)
- **--model, -m** — путь к файлу модели (по умолчанию: model/tle\_model.pth)
- **--normalizer, -n** — путь к нормализатору (по умолчанию: model/normalizer.pkl)
- **--history** — файл истории TLE (по умолчанию: data/tle\_history.json)
- **--output, -o** — файл для сохранения JSON отчёта
- **--only-anomalies, -a** — показывать только аномалии
- **--cpu** — принудительно использовать CPU (по умолчанию: автоопределение)
### **5.3.1. Примеры команд**
*Анализ всех объектов в файле:*

python main.py -f data/geo\_satellites.txt

*Показать только аномалии:*

python main.py -f data/test.txt --only-anomalies

*Сохранить результат в JSON:*

python main.py -f data/test.txt -o results.json
## **5.4. Формат TLE файла**

# **6. Структура отчёта**
Для каждого объекта система генерирует детальный отчёт, включающий следующие разделы:
## **6.1. Классификация объекта**
- Тип объекта (спутник, мусор, ступень ракеты, осколок)
- NORAD ID (каталожный номер)
- COSPAR ID (международное обозначение)
- Статус по базе данных
## **6.2. Статус объекта**
- Операционный статус (активный, неактивный, потерянный)
- Уверенность в статусе [%]
- Обоснование статуса
## **6.3. Орбитальные параметры**
- Тип орбиты (GEO, MEO, LEO и т.д.)
- Период обращения [минуты, часы]
- Большая полуось [км]
- Высота апогея и перигея [км]
- Все 6 входных параметров
- Возраст данных [дни]
## **6.4. Оценка модели**
- Ошибка реконструкции
- Уверенность модели [%]
- Надежность (высокая/средняя/низкая)
- Оценка аномальности [0-1]
- Статус (норма/аномалия)
- Факторы, влияющие на качество оценки
## **6.5. Детальный анализ аномалии**
Если обнаружена аномалия:

- Диагноз — какой параметр вызвал аномалию
- Топ-3 параметра с наибольшим отклонением
- Абсолютная и относительная ошибка для каждого
## **6.6. Рекомендуемые углы коррекции**
*Только для спутников:* детальные расчёты требуемых маневров
## **6.7. Анализ деградации орбиты**
*Для космического мусора:* прогноз снижения орбиты и входа в атмосферу
## **6.8. Оценка расхода топлива**
*Для спутников:* требуемый Delta-V с разбивкой по типам коррекции
## **6.9. Риски**
- Уровень риска опасного сближения
- Предупреждения
## **6.10. Итоговые рекомендации**
Контекстно-зависимые рекомендации с приоритетами:

- **КРИТИЧНО** — требует немедленных действий
- **ВЫСОКИЙ ПРИОРИТЕТ** — рекомендуется выполнить в ближайшее время
- **СРЕДНИЙ ПРИОРИТЕТ** — планируется на будущее
- **ИНФОРМАЦИЯ** — справочная информация

# **9. Ограничения и известные проблемы**
## **9.1. Ограничения модели**
- **Специализация на GEO орбитах**: модель обучена преимущественно на геостационарных спутниках. Для LEO/MEO точность может быть ниже.
- **Зависимость от качества TLE**: устаревшие данные (>7 дней) снижают точность анализа.
- **Отсутствие внешних факторов**: модель не учитывает солнечную активность, гравитационные возмущения от Луны/Солнца.
## **9.2. Рекомендации по улучшению**
- Регулярно обновлять обучающий набор данных
- Использовать свежие TLE (не старше 3-5 дней)
- Проводить периодическое переобучение модели
- Валидировать результаты с помощью независимых источников
# **10. Заключение**
Система анализа орбитальных данных TLE представляет собой мощный инструмент для автоматизированного мониторинга космических объектов. Использование методов глубокого обучения позволяет эффективно детектировать аномалии и предоставлять операторам спутников детальные, контекстно-зависимые рекомендации по управлению орбитой.

Ключевые преимущества системы:

- **Полностью автоматизированный анализ** — от парсинга до генерации отчёта
- **Высокая точность детекции аномалий** благодаря автоэнкодеру
- **Исторический анализ** для выявления трендов
- **Практические рекомендации** с расчётом углов и Delta-V
- **Оценка рисков** столкновения и схода с орбиты

Система активно развивается и может быть адаптирована для решения специфических задач космической отрасли.
